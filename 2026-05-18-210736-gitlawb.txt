● マージする必要はありません。ブランチを独立させればいいです。

  master (GitHub) ──→ git push origin master
    └─ README.github.md
    └─ 通常のコード

  gitlawb (gitlawb) ──→ git push gitlawb gitlawb:main
    └─ README.gitlawb.md
    └─ 通常のコード（同じ）

  実運用

  # 1. 通常の開発はmasterで
  git checkout master
  # ... コード変更 ...
  git commit -am "feature X"
  git push origin master        # GitHubにpush

  # 2. gitlawbにも同期したいとき
  git checkout gitlawb
  git merge master              # masterの変更を取り込む
  git push gitlawb gitlawb:main # gitlawbにpush

  逆にgitlawbだけのファイル

  git checkout gitlawb
  # gitlawb専用ファイルを追加
  git commit -am "gitlawb only docs"
  git push gitlawb gitlawb:main
  # masterには影響しない

  要点：
  - master → GitHub用
  - gitlawb → gitlawb用
  - コードの同期は git merge で片方向に流す
  - マージ不要なファイルは各ブランチにだけ置く

  これで両方独立に管理できます。

❯  --expose-gc を付けていても、スピナーの問題は変わりません。                   
                                                                                
                                                                                
  --expose-gc の効果:                                                           
                                                                                
  - JavaScriptから global.gc() を呼べるようになる                               
  - 手動でGCをトリガーできる                                                    
                                                                                
                                                                                
  しかし:                                                                       
                                                                                
  - スピナーは120msごとに新しいオブジェクトを生成し続ける                       
  - gc() を呼んでも、すぐにまた埋まる                                           
                                                                                
  - 蛇口が開いたまま排水口を掃除しているだけ                                    
                                                                                
                                                                                
  むしろ危険:                                                                   
                                                                                
  - --expose-gc で頻繁にGCを呼むと、GCのオーバーヘッドでCPU使用率がさらに上がる 
                                                                                
  - スピナーの描画 + GCの両方でCPUを食う

は関係ない。根本対策はQueryGuardのタイムアウト（今回実装済み）です。

  1. コストが低い — 警告出力だけなので数行の変更                                
                                                                                
  2. デバッグ時の手がかり — 「なぜGCが効かないのか」の原因特定が早くなる        
  3. nheap_limitのJSコールバックがgc()を呼べる — --expose-gc があれば更有効
