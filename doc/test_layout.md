# OpenClaude / Ink CLI レイアウト確認テスト (nproxy transform/strip-ansi)

## テスト対象

```bash
node -r ./node/nproxy.js /usr/bin/openclaude
NPROXY_TEXT=transform node -r ./node/nproxy.js /usr/bin/openclaude
NPROXY_TEXT=strip-ansi node -r ./node/nproxy.js /usr/bin/openclaude
```

## 環境変数

| 変数 | テスト値 | 備考 |
|------|----------|------|
| NPROXY_TEXT | passthrough / transform / strip-ansi | デフォルト passthrough |
| NPROXY_MONITOR | 1 (default) | チャンク分割 + メモリ監視有効 |
| NPROXY_DEBUG | 1 | デバッグログ出力 |

## テスト項目

### 1. ASCIIアート / 箱描画
**期待**: 連続行で崩れずに表示される
**テスト**: `print ascii art` や `print Nproxy logo`
**確認**: 各行の改行が正しく保持され、ボックス描画の罫線が揃っている

### 2. コマンド一覧ポップアップ
**期待**: スラッシュ `/` で一覧がポップアップ、整列して表示
**テスト**: `/` を入力
**確認**: リストの各行が整列、選択可能、枠線が崩れていない

### 3. 応答インジケータ (スピナー)
**期待**: `* Slithering…` / `* Thinking...` が `\r` で上書きされる
**テスト**: 長い応答を待つ
**確認**: インジケータが1行に固定され、改行が増えない

### 4. 改行送信 (iPhone / モバイル)
**期待**: 1回の改行 (Enter) で送信、2回改行不要
**テスト**: モバイルブラウザから入力 → 改行
**確認**: メッセージが二重送信されない、空行が余計に入らない

### 5. 色表示
**期待**: カラーSGRシーケンスが正しく出力される
**テスト**: `/help` やエラーメッセージ、シンタックスハイライト
**確認**: 色再現が崩れていない（strip-ansi では色が消えてもレイアウトは正常）

### 6. 大量出力 / フレーム描画
**期待**: 大量テキスト出力時にスクロールが崩れない
**テスト**: 大きなプロンプト応答、ログ出力
**確認**: スクロール正常、古い出力が途中で切れない、フレーム同期が維持される

### 7. カーソル表示
**期待**: OpenClaude 終了後にカーソルが表示される
**テスト**: 終了 → `echo` で確認
**確認**: `?25h` (show) が正しく出力され、カーソルが消えたままにならない

### 8. チャンク分割 (256KB)
**期待**: 分割時のレイアウト崩れがない
**テスト**: 750KB 程度のテキストを貼り付け（吾輩は猫である 等）
**確認**: 出力が連続して表示され、分割による不自然な改行や遅延がない

### 9. メモリ状態遷移
**期待**: pressure/critical で自動モード切替しても出力が止まらない
**テスト**: 大量のデータを流し続けてメモリを圧迫
**確認**: モード切替後も Ink のレイアウトが崩れない

## 既知の問題

- **transform モード**: タイムスタンプ挿入により Ink の box-drawing が崩れることが確認済み。passthrough を推奨。
- **strip-ansi モード**: 色が削除されるがレイアウト自体は維持される。
- **OSC 8 ハイパーリンク**: `screen` の OSC 8 不可対応として `doc/08_principles.md` 参照。
