5. メモリ消費 (RSS)

dd (100MB /dev/zero) を各モードでパイプした際の最大 RSS。Rust 版は全モードで約 5MB 未満。

モード最大 RSSNode.js 比
--text=off4,620 KB約 1/10
--text=passthrough5,136 KB
--text=transform4,708 KB

※ Node.js v18 は同条件約 40-50 MB。

※ transform モードの内部バッファは 1MB 単位で分割流出。改行のない /dev/zero のようなデータでも RSS 肥大なし。

7. 今後の課題

  1. エラーハンドリング: 現状 unwrap が多い。anyhow/thiserror 導入によるエラー伝搬の明確化。
  2. リングバッファ設定: --ring-size 等の CLI オプション追加。
  3. OOM テスト: メモリポリシー強制終了の実機確認。
  4. CI/CD: GitHub Actions でのテスト・ベンチマーク自動実行。
  5. Windows 対応: procfs 非依存のメモリ取得。
